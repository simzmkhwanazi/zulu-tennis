Data Model

One consistent schema behind every worksheet, scouting sheet, dashboard and the improvement engine — so your data connects instead of scattering across notebooks. Designed as if a future version could become an app.

Why a single source of truth

You already track scores, opponents, fitness and goals — but in separate places that never talk to each other. The Data Model defines shared entities (tables) and how they relate, so a match feeds analytics, analytics feed the Improvement Engine, and that feeds your roadmap. Every printable sheet in this system names the entity it maps to.

🎯 Purpose

Make "Why did I lose that point / that match / last quarter?" answerable from one connected record. Level B

⚠️ Common mistakes
  • Tracking everything in one giant note — no links, so nothing aggregates.
  • Logging data you never review. Only track fields that feed a decision (analytics → priorities → roadmap).
✅ Success metrics
  • Each match record links to one opponent and produces one KPI set.
  • Every printable sheet you use names the entity it maps to.
  • Your top improvement priority is traceable back to specific matches/points.
